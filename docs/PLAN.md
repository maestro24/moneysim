# 머니시뮬 (MoneySim) — 설계 문서

몬테카를로 재테크 시뮬레이터. "몇 살에 얼마" 단정 대신 **성공확률과 백분위 밴드**로 답하는 5종 도구 + 허브.

## 핵심 원칙

1. **수학 정확성 게이트** — 닫힌형 공식(복리·연금·상환)과 몬테카를로 수렴치를 Node 테스트로 교차 검증. 시드 고정 RNG(mulberry32)로 100% 재현 가능.
2. **확률로 말한다** — 성공확률·10/50/90 백분위 밴드를 시각화. 단일 수익률 가정 계산기와의 차별점.
3. **투자 조언 아님** — 전 페이지 하단에 고지 상시 노출. 특정 종목·상품 언급 금지.
4. **의존성 0** — 외부 라이브러리 없이 Canvas 차트 직접 구현. 빌드 0, 유지비 0원.

## 제약 조건

- GitHub Pages 정적 호스팅 단독. Vanilla JS ES modules. 라이트 테마 전용. 한국어 UI.
- 몬테카를로(10,000 시나리오)는 module Web Worker에서 실행, 1,000회마다 진행률 postMessage.
- 원시 경로 전송 금지 — Worker 안에서 백분위·성공률·히스토그램으로 집계 후 전달.
- 반올림·원화 표시는 UI 층(js/ui/format.js)에서만. 코어는 부동소수 유지.
- 파일당 800줄 미만, 기능별 소형 모듈.

## 수학 규약

- 월 이율 변환 2규약을 명시적으로 분리: 대출 = 명목 r/12, 투자 = 실효 (1+r)^(1/12)−1.
- GBM: r = exp(μ_log + σ·z) − 1, μ_log = ln(1+μ) − σ²/2 → 시나리오 산술평균 = μ.
- 엔진은 월 이산화(μ_log/12, σ/√12) 사용 → E[1+r_m] = (1+μ)^(1/12)이 정확히 성립,
  월 적립·인출의 기대값이 닫힌형과 일치 (수렴 테스트의 근거).
- σ=0이면 결정론적으로 닫힌형과 완전 일치 (테스트로 보증).
- 현금흐름은 월말(기말불) 규약. 인출액은 현재가치 입력 → t=0부터 월 물가 반영 증액.
